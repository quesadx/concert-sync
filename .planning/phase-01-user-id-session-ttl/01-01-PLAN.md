## Phase Goal

**As a** concertgoer, **I want to** identify myself with a display name and have all my selected seats share one expiration timer, **so that** I can select multiple seats without each expiring independently.

<objective>
Establish the user identification and session-TTL foundation so that a concertgoer can log in, identify themselves, and reserve a single seat within a session-based TTL framework.

**Purpose:** This is the vertical-slice foundation — after this plan, a user can enter their name in a TUI, connect to the server, and reserve a seat. The server creates a session for them, returns the session_id as transaction_id, and manages TTL at session level.

**Output:** LoginScreen UI, SessionManager/UserSession classes, user_id injection, protocol validation, session-aware RESERVE handler, and a failing-then-passing e2e test.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | ConcertClient → ConcertServer | accept | User ID is self-asserted in Phase 1. Full auth is explicitly out of scope per PROJECT.md. Server does not validate identity beyond string presence. |
| T-01-02 | Tampering | SessionManager.get_or_create | mitigate | `get_or_create` uses `threading.Lock` around the check-and-create block to prevent race conditions where two threads create two sessions for the same user_id (per RESEARCH.md Pitfall 1). |
| T-01-03 | Information Disclosure | protocol_validator.validate_request | mitigate | user_id presence check returns generic "Missing required field: user_id" message — does not reveal valid user IDs or session state. |
| T-01-SC | Tampering | All imports in new/modified files | mitigate | No new external packages. All imports are from stdlib (threading, time, uuid, dataclasses, typing) or existing project modules. Slopcheck will verify no untrusted code was introduced. |
</threat_model>
