## Spacing Scale

**Textual CSS system:** Spacing is in integer "cells" (terminal character grid units). The existing `styles.tcss` establishes these values:

| Token | Value (cells) | Terminal Equiv. | Usage | Source |
|-------|---------------|-----------------|-------|--------|
| xs | 0 | ~0px | No spacing, edge-to-edge elements | Implicit |
| sm | 1 | ~8px (variable) | Default element spacing: `padding: 1`, `margin-bottom: 1` | `styles.tcss:15,37` |
| md | 2 | ~16px | Section padding within panels | Frequent in codebase |
| lg | 3 | ~24px | Compound margins | Occasional |

**Exceptions for Phase 1:**
- **LoginScreen** uses `Screen` default layout (vertical, centered) — no custom spacing needed
- **New elements** (LoginScreen widgets): follow existing `Input`, `Button` spacing from `styles.tcss` — `width: 100%; margin-bottom: 1`
- **No spacing overrides needed** for Phase 1

## Typography

**Constraint:** Textual TUI runs in the user's terminal. Font family, size, and rendering are **terminal-controlled**, not app-controlled.

| Role | Size | Weight | Styling | Terminal Control |
|------|------|--------|---------|------------------|
| Body | terminal default | terminal default (400 equiv) | regular | Uncontrollable |
| Panel Title | terminal default | **bold** (via `text-style: bold`) | color: `#9ad4d6` | Uncontrollable, CSS sets bold flag |
| Label / Legend | terminal default | terminal default | color: `#9fb3c8` / `#b9cbe0` | Uncontrollable, CSS sets color |
| Header | terminal default | bold (Textual default) | — | Textual `Header` widget default |
| Button | terminal default | bold (Textual default for `variant="primary"`) | — | Textual default styling |
| Status Line | terminal default | terminal default | color: `#d8e6f5` | `styles.tcss:137-139` |
| Seat Map Cell | terminal default | terminal default | color: `#e0f0ff` | Uncontrollable, CSS sets color |

**Line height:** Not controllable in Textual TUI — terminal renders at 1.0 (character grid).

**Phase 1 additions:**
- LoginScreen welcome label: **bold** with color `#9ad4d6` (matching existing `.panel-title` pattern)
- LoginScreen prompt label: terminal default weight, color `#d8e6f5` (matching existing `.status-line` pattern)
- LoginScreen Input: standard `Input` styling from `styles.tcss:33-38`
- LoginScreen Button ("Join"): `variant="primary"` — Textual applies default primary styling
- App title after login: `ConcertSync — {user_id}` — shown in `Header(show_clock=True)` widget

## Copywriting Contract

All copy for Phase 1's new LoginScreen, as designed in RESEARCH.md:

| Element | Copy | Context |
|---------|------|---------|
| **Primary CTA** | *"Join"* | Button on LoginScreen to submit display name |
| **Welcome heading** | *"Welcome to ConcertSync"* | Top of LoginScreen, bold, accent color |
| **Prompt label** | *"Enter your display name to join:"* | Below welcome, guides user action |
| **Input placeholder** | `e.g., Alice` | In the name Input widget, before user types |
| **Empty name error** | *"Name cannot be empty! Enter your display name:"* | Error shown when user clicks Join with empty input — replaces welcome label text |
| **App title after login** | `ConcertSync — {user_id}` | Set via `self.title` on ConcertTextualApp after LoginScreen dismisses. Replaces existing `ConcertSync Terminal Console` |
| **Destructive actions** | **None** in Phase 1 | Cancel (existing) uses `variant="warning"` with no new confirmation dialog. No new destructive actions introduced. |

**Error state behavior (LoginScreen):**
1. User clicks "Join" with empty input → welcome label updates to show error text (in-place, no popup)
2. User types and submits via Enter → same validation runs
3. No destructive confirmations needed in Phase 1

## Screen Flow

```text
┌────────────────────────────────────────┐
│  App Startup                            │
│  ConcertTextualApp.on_mount()           │
│  Push LoginScreen                       │
└────────────────┬───────────────────────┘
                 │
                 ▼
┌────────────────────────────────────────┐
│  LoginScreen                            │
│  ┌────────────────────────────────────┐│
│  │  Header (shows clock)              ││
│  │                                     ││
│  │  "Welcome to ConcertSync" (bold)   ││
│  │  "Enter your display name to join:" ││
│  │  [            Input            ]   ││
│  │  [          Join (primary)     ]   ││
│  │                                     ││
│  │  Footer                             ││
│  └────────────────────────────────────┘│
└───────────────┬────────────────────────┘
                │ User enters name + presses Join / Enter
                ▼
┌────────────────────────────────────────┐
│  LoginScreen.dismiss(user_id)          │
│  → App._on_login(user_id)              │
│     - self.user_id = user_id           │
│     - self.client = ConcertClient(user_id)│
│     - self.title = "ConcertSync — Alice"│
│     - Connect to server                │
│     - Proceed to main dashboard        │
└────────────────────────────────────────┘
```

## Interaction Contract

| Interaction | Event | Behavior | Source |
|-------------|-------|----------|--------|
| **App mounts** | `on_mount()` | Push `LoginScreen`. Do NOT connect to server yet. | RESEARCH.md:538-539 |
| **User types name** | Input value changes | No validation on keystroke. Only submit validates. | RESEARCH.md:518-527 |
| **User presses "Join"** | `Button.Pressed` on `#join-btn` | `_submit_name()`: trim whitespace → if non-empty, `dismiss(name)` → if empty, update welcome label to error text | RESEARCH.md:518-527 |
| **User presses Enter** | `Input.Submitted` on `#name-input` | Same as Join button: `_submit_name()` | RESEARCH.md:521-523 |
| **LoginScreen dismissed** | `dismiss(user_id)` → app `_on_login(user_id)` | Store user_id, create ConcertClient with user_id, update app title, connect to server, proceed to dashboard | RESEARCH.md:541-545 |
| **User quits** | `q` key binding | Existing quit behavior — no change in Phase 1 | `app.py:84` |
