# Quick Task 260604-r7u: Fix per-seat TTL tracking

**Goal:** Each reserved seat (or batch) gets its own independent TTL. When TTL expires, only that seat is released — not all seats in the session. Fix race conditions in the monitor expiry double-check.

## Root Cause

`UserSession` uses a single `last_activity` timestamp for ALL seats. When `reset_ttl()` is called (every RESERVE/RESERVE_BATCH), it refreshes the TTL for all previously reserved seats, making their individual TTLs "disappear."

Additionally, the monitor's `expire_session()` double-check only verifies `state == ACTIVE` but not `is_expired`, creating a race window where a TTL-refreshed session can be prematurely expired.
