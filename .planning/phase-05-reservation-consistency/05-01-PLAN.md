<objective>
Add a RESERVE_SELECTED server handler and client method that atomically reserves a list of seats in a single transaction, enabling the TUI to batch multiple individual clicks into one atomic operation.

Purpose: Bridges the gap between individual seat clicks (each a separate RESERVE) and true atomic multi-seat reservation. Instead of N individual RESERVE calls that can interleave with other sessions, the TUI sends one RESERVE_SELECTED for the user's chosen seats.
Output: RESERVE_SELECTED handler + ConcertClient.reserve_selected()
</objective>
