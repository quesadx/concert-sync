<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GUI thread ↔ Worker thread | Qt signal-slot boundary — data crosses from background thread to main thread via Signal.emit() |
| User click → seat selection | User input enters the application through QTableWidget cell click — must validate seat state before toggling |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | Network workers blocking GUI thread | mitigate | All network calls run on daemon `threading.Thread` (not main thread); `run_worker` ensures non-blocking dispatch; QTimer continues polling even if workers stall |
| T-03-02 | Tampering | User clicks non-AVAILABLE seat | mitigate | `_on_cell_clicked` reads server state from `Qt.UserRole` data; MainWindow slot checks state before processing; non-AVAILABLE seats show status message only |
| T-03-03 | Information Disclosure | Event log displays transaction IDs | accept | Transaction IDs (UUIDs) are shown in event log — these are opaque identifiers, not PII; educational project with no auth system |
</threat_model>
