    Modify `_reserve_pending_selections()` to insert the check after the "pending_selections empty" guard and before the "ensure_client" call:

    **Before the `try/except` block that calls `self._ensure_client()`, add:**

    ```python
    # Saturated zone pre-flight check: verify pending seats against latest snapshot
    conflict_seats = []
    for seat in self.pending_selections:
        section = seat["section"]
        row = seat["row"]
        col = seat["col"]
        grid = self.seat_map_snapshot.get(section, [])
        if row < len(grid) and col < len(grid[row]) and col >= 0 and row >= 0:
            if grid[row][col] != "AVAILABLE":
                conflict_seats.append(seat)

    if conflict_seats:
        conflict_msg = ", ".join(
            f"{s['section']}({s['row']},{s['col']})" for s in conflict_seats
        )
        self.pending_selections = [
            s for s in self.pending_selections
            if s not in conflict_seats
        ]
        self._render_seat_map()
        warning = f"Conflict: {conflict_msg} no longer available — removed from pending"
        self._set_status(warning)
        self._append_event(f"[SATURATED] {warning}")
        if not self.pending_selections:
            return
    ```

    This check:
    - Compares each pending seat against the latest `seat_map_snapshot` (refreshed every 1s by _refresh_every_second)
    - Removes any seat that is no longer AVAILABLE from `pending_selections`
    - Shows a clear warning message listing the conflict seats
    - Re-renders the seat map to reflect the removed pending selections
    - Returns early if all pending seats were removed
    - Makes **zero** server-side changes (SAT-02 preserved)
    - Is best-effort: a TOCTOU window exists between the snapshot and the server-side RESERVE_SELECTED, but this catches obvious conflicts from other users' reservations that occurred before the latest refresh
    - Uses dict equality (`s not in conflict_seats`) — Python dicts compare by value, so this works correctly for the pending_selections entries

    The existing flow after this check (ensure_client, seat_summary, thread launch, worker, succeeded/failed) remains completely unchanged.
