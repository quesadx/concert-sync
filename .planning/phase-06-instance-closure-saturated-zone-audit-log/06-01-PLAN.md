```python
class GlobalLog:
    def __init__(self, filepath="logs/system.log"):
        self.filepath = Path(filepath)
        self.filepath.parent.mkdir(parents=True, exist_ok=True)
        self.mutex_log = threading.Lock()

    def append(self, event_type, message):
        with self.mutex_log:
            timestamp = datetime.datetime.now().isoformat()
            log_entry = f"[{timestamp}] [{event_type}] {message}\n"
            with self.filepath.open("a", encoding="utf-8") as f:
                f.write(log_entry)
```

    **B. Add _release_all_sessions() method:**

    Add a new private method after `_cleanup_stale_reservations()` (after line 74). The method should:

    1. Get a snapshot of all ACTIVE sessions via `self.session_manager.get_all_active()`
    2. For each session, group seats by section using `defaultdict(list)`
    3. Acquire `self.mutex_manager.table_and_sections(sections_list)` — same lock hierarchy as `expire_session` in monitor_thread.py
    4. Inside the lock:
       a. Re-check that the session is still ACTIVE via `self.session_manager.get_by_session_id(session.session_id)` (race with in-flight TransactionalThread)
       b. For each section: iterate seats, if `seat_matrix.seats[section][row][col] == SeatState.RESERVED`, set to `SeatState.AVAILABLE`
       c. Remove session via `self.session_manager.remove(session.user_id)`
       d. Release semaphore slots via `self.semaphore_mgr.release_multiple(section, count)`
    5. Track total released count; if > 0, log: `self.global_log.append("SHUTDOWN", f"Released {total} seat(s) from {len(sessions)} session(s)")`

    Key design points:
    - The lock is acquired per-session (not globally for all sessions at once), so each session's seats are released atomically but sessions are released sequentially. This prevents acquiring all section locks simultaneously (which could block).
    - `get_by_session_id()` double-check inside the lock prevents releasing a session already CONFIRMED/CANCELLED/EXPIRED by a concurrent thread.
    - The method is called after socket close + small sleep — no new connections arrive, no new TransactionalThreads start. Existing TransactionalThreads may still run briefly.
    - Import `defaultdict` and `SeatState`/`ReservationStatus` at the top of the file.
